# Testing Skill

## Objectives

Ensure:
- correctness
- reliability
- regression protection
- maintainability
- confidence in deployments

---

# Testing Principles

Prefer:
- deterministic tests
- isolated tests
- fast feedback
- edge-case validation
- meaningful assertions

Avoid:
- brittle tests
- excessive mocking
- duplicated tests
- implementation-coupled tests

---

# Required Outputs

Generate:
- unit tests
- integration tests
- edge case scenarios
- validation strategies
- test coverage recommendations

---

# Workflow

1. Analyze functionality
2. Identify critical paths
3. Identify edge cases
4. Generate tests
5. Validate behavior
6. Verify failures
7. Refactor tests if necessary

---

# Validation Areas

Always consider:
- invalid inputs
- concurrency
- async behavior
- security boundaries
- error handling
- retries/timeouts
- performance-sensitive paths

---

# Examples

Use this skill when:
- validating features
- creating automated tests
- improving reliability
- investigating regressions
