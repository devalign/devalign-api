# Review Skill

## Objectives

Review code for:
- maintainability
- readability
- architecture consistency
- scalability
- security
- performance
- technical debt

---

# Review Areas

Analyze:
- architecture violations
- coupling
- code smells
- duplication
- unclear naming
- missing abstractions
- error handling
- security risks
- performance bottlenecks
- testing quality

---

# Rules

- Be critical but constructive
- Explain WHY something is problematic
- Suggest realistic improvements
- Avoid unnecessary refactors
- Prioritize maintainability

---

# Workflow

1. Understand feature purpose
2. Review architecture alignment
3. Analyze module boundaries
4. Identify risks
5. Detect code smells
6. Validate standards
7. Suggest improvements
8. Summarize findings

---

# Output Format

Always include:
- Findings
- Severity
- Risks
- Suggested Fixes
- Refactor Recommendations

---

# Examples

Use this skill when:
- reviewing pull requests
- auditing systems
- refactoring
- improving code quality
- reducing technical debt
