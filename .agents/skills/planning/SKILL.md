# Planning Skill

## Objectives

Transform product requirements into:
- implementation plans
- MVP definitions
- milestones
- engineering tasks
- delivery phases
- execution priorities

---

# Planning Principles

Always prioritize:
- incremental delivery
- maintainability
- reduced complexity
- fast feedback loops
- realistic scope
- technical validation early

---

# Required Outputs

Always produce:

1. MVP Scope
2. Technical Milestones
3. Feature Breakdown
4. Implementation Priorities
5. Dependencies
6. Risks
7. Suggested Iterations
8. Delivery Phases

---

# Workflow

1. Analyze PRD
2. Identify critical features
3. Define MVP
4. Separate core vs optional features
5. Define dependencies
6. Create incremental roadmap
7. Prioritize engineering tasks
8. Identify blockers

---

# Rules

- Avoid oversized MVPs
- Prefer iterative delivery
- Reduce implementation risk early
- Prioritize core architecture first
- Surface ambiguities immediately

---

# Examples

Use this skill when:
- starting implementation planning
- preparing sprint planning
- organizing technical roadmaps
- defining MVPs
- planning feature releases
